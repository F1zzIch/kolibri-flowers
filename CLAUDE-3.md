## 1. О проекте

Сайт-витрина для цветочного магазина **_kolibri_flowers_** из города **Мозырь** (Беларусь),
который сейчас продаёт через Instagram. Задача сайта — красивый онлайн-каталог букетов
и композиций, который усиливает инстаграм-присутствие и позволяет клиентам быстро
посмотреть ассортимент, цены и оформить заказ.

**Ключевые требования заказчика:**

1. Красивый, «вкусный» дизайн уровня премиальной флористики.
2. Плавные, аккуратные анимации (не перегруженные, но заметные).
3. Администратор магазина может **сам редактировать каталог** через защищённую
   админ-панель: добавлять/менять/удалять букеты, цены, фото и категории — без программиста.

**Аудитория:** в основном мобильные пользователи из Instagram. Поэтому проект
**mobile-first**: сначала отлично выглядит на телефоне, потом на десктопе.

## 2. Технологический стек

Используй именно этот стек, если нет явной причины отступить:

| Слой | Технология | Зачем |
|------|------------|-------|
| Фреймворк | **Next.js (App Router) + TypeScript** | SSR/SSG, хороший SEO, удобный роутинг |
| Стили | **Tailwind CSS** | быстрая стилизация, единая дизайн-система |
| Анимации | **Framer Motion** | плавные переходы, scroll-reveal, hover-эффекты |
| Иконки | **lucide-react** | лёгкие SVG-иконки |
| База данных + Auth + Хранилище фото | **Supabase** (Postgres + Auth + Storage) | каталог, вход для админа, загрузка картинок — всё в одном, есть бесплатный тариф |
| Деплой | **Vercel** | бесплатно, интеграция с Next.js |

**Почему Supabase:** заказчику нужна возможность редактировать каталог. Supabase даёт
готовую базу, авторизацию (логин только для админа) и хранилище для фотографий букетов.
Никаких собственных серверов поднимать не нужно.

> Если у заказчика нет аккаунта Supabase — на этапе разработки используй мок-данные
> из локального JSON (`src/data/mock-products.ts`), но архитектуру строй сразу под Supabase,
> чтобы переключение было лёгким.

## 3. Визуальный стиль

Название «Колибри» → лёгкость, изящество, природа. Стиль — нежный, премиальный,
с воздухом и хорошей типографикой.

**Цветовая палитра (CSS-переменные в `globals.css`):**

```
--bg:        #FBF7F2   /* тёплый кремовый фон */
--surface:   #FFFFFF   /* карточки */
--ink:       #2B2B2B   /* основной текст */
--muted:     #8A8178   /* второстепенный текст */
--accent:    #E08AA0   /* нежно-розовый акцент (кнопки, ссылки) */
--accent-2:  #1F6E5E   /* изумрудно-зелёный (колибри, детали) */
--gold:      #C9A86A   /* тонкие премиальные акценты */
```

**Типографика:**
- Заголовки — изящный serif-шрифт (например, *Playfair Display* или *Cormorant*).
- Текст — чистый sans-serif (*Inter* / *Manrope*).
- Много «воздуха», крупные отступы, мягкие тени, скруглённые углы (`rounded-2xl`).

**Логика интерфейса:**
- Каталог — сетка карточек с крупными фото квадратного/портретного формата (как в Instagram).
- Карточка: фото, название, цена, бейдж «Хит» / «Нет в наличии», кнопка «Заказать».
- Всё кликабельно, переходы плавные, ничего резкого.

## 4. Анимации (важная часть!)

Используй Framer Motion. Анимации должны быть **плавными и сдержанными**, не «прыгающими».

Обязательный набор:

1. **Появление при скролле** (scroll-reveal): секции и карточки плавно всплывают
   снизу + fade-in при попадании во вьюпорт (`whileInView`, `viewport={{ once: true }}`).
2. **Hover на карточках:** лёгкое увеличение фото (`scale 1.04`), мягкая тень,
   проявление кнопки «Заказать».
3. **Hero-секция:** плавная анимация заголовка по словам/буквам + тонкий
   фоновый эффект (медленный градиент или парящие лепестки/силуэт колибри).
4. **Переходы между страницами** и открытие карточки товара (модалка) — fade + scale.
5. **Stagger** при появлении сетки каталога (карточки появляются по очереди с задержкой).
6. **Микроанимации кнопок:** `whileHover` / `whileTap`.

Технические правила:
- Уважай `prefers-reduced-motion` — отключай тяжёлые анимации для тех, кто их не хочет.
- Длительности 0.3–0.6с, easing мягкий (`easeOut`).
- Не анимируй то, что бьёт по производительности на телефоне.

## 5. Структура сайта

**Публичная часть:**

- `/` — Главная: Hero → краткое о магазине → популярные букеты → отзывы/Instagram → контакты.
- `/catalog` — Каталог: фильтр по категориям, сетка карточек, поиск (по желанию).
- `/catalog/[slug]` — Страница (или модалка) букета: галерея фото, описание, цена, состав, кнопка заказа.
- `/about` — О магазине (можно объединить с главной).
- `/contacts` — Контакты: телефон, Instagram, Telegram/WhatsApp, адрес и карта Мозыря, часы работы.

**Заказ:** отдельной корзины и онлайн-оплаты НЕ нужно (это витрина).
Кнопка «Заказать» ведёт в Instagram Direct (`https://instagram.com/_kolibri_flowers_`)
или в Telegram/WhatsApp с предзаполненным текстом «Здравствуйте! Хочу заказать букет: <название>».

**Закрытая часть (админка):**

- `/admin/login` — вход (Supabase Auth, только для администратора).
- `/admin` — список товаров, кнопки добавить/редактировать/удалить.
- `/admin/products/new` и `/admin/products/[id]` — форма товара с загрузкой фото.
- `/admin/categories` — управление категориями.
- `/admin/settings` — контакты магазина, ссылки на соцсети, часы работы.

Все `/admin/*` маршруты защищены: без авторизации редирект на `/admin/login`.

## 6. Модель данных (Supabase)

**Таблица `categories`:**
```
id          uuid (pk)
name        text         -- «Авторские букеты», «Розы», «Композиции в коробке» ...
slug        text unique
sort_order  int
```

**Таблица `products`:**
```
id           uuid (pk)
name         text
slug         text unique
description  text
price        numeric        -- цена в BYN
category_id  uuid (fk -> categories.id)
images       text[]         -- массив URL из Supabase Storage
is_available boolean        -- в наличии / нет
is_featured  boolean        -- показывать на главной как «хит»
sort_order   int
created_at   timestamptz default now()
```

**Таблица `settings`** (одна строка):
```
phone         text
instagram_url text
telegram_url  text
whatsapp_url  text
address       text          -- адрес в Мозыре
working_hours text
```

**Storage bucket** `product-images` — для фотографий букетов.

**Row Level Security (RLS):**
- Публичное чтение `products`, `categories`, `settings` — разрешено всем.
- Запись/изменение/удаление — только аутентифицированному админу.
- Загрузка в `product-images` — только админу; чтение публичное.

## 7. Админ-панель: требования

Это ключевая фича — заказчик должен управлять каталогом сам.

- **Вход** по email + паролю (один администратор). Можно завести через Supabase заранее.
- **Список товаров** с миниатюрой, названием, ценой, категорией, статусом наличия.
- **Добавить / редактировать товар:** название, описание, цена, категория,
  загрузка нескольких фото (drag-and-drop), флажки «в наличии» и «хит».
- **Удалить товар** с подтверждением.
- **Категории:** добавить/переименовать/удалить, изменить порядок.
- **Настройки:** телефон, ссылки на Instagram/Telegram/WhatsApp, адрес, часы работы.
- Интерфейс админки — простой и понятный для непрограммиста, на русском языке,
  с понятными подсказками и сообщениями об успехе/ошибке (toast).
- После сохранения данные сразу видны на публичном сайте.

## 9. Соглашения по коду

- **TypeScript строгий**, без `any` без необходимости.
- Компоненты функциональные, маленькие и переиспользуемые.
- Тексты интерфейса — на **русском языке**.
- Цвета и отступы — через токены Tailwind / CSS-переменные, без «магических» хардкодов.
- Все изображения — через `next/image` с правильными размерами и `alt`.
- Доступность: семантический HTML, контраст текста, фокус-стейты, alt у картинок.
- Секреты только в `.env.local`, в репозиторий не попадают.
- Понятные имена компонентов и коммитов.
